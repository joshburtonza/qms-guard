## Phase 2 Planning (documented 2026-03-12)

### NC Submission — Field Analysis

**Required fields (Zod-validated — blocking on submit):**
| Field | Type | Notes |
|---|---|---|
| `department_id` | select | Fetched from `departments` table |
| `site_location` | text | Free text, min 1 char |
| `shift` | enum | day / night / general |
| `date_occurred` | date | Defaults to today |
| `source` | enum | 8 values + 'other' (requires `source_other`) |
| `category` | enum | 7 values + 'other' (requires `category_other`) |
| `severity` | enum | critical / major / minor |
| `description` | textarea | Min 50 characters |
| `responsible_person` | select | Fetched from active `profiles` |
| `due_date` | date | Defaults to +30 days |

**Optional fields (not in schema — non-blocking):**
| Field | Notes |
|---|---|
| `applicable_clauses` | Array of ISO clause strings. Labelled "(optional)" in UI. Defaults to `[]`. |
| Attachments | Up to 5 files (jpg/png/pdf/doc/docx, max 10MB each). Uploaded post-insert to `nc-attachments` storage. |

**Programmatically set on submit:**
- `reported_by` — from auth profile
- `status` — hardcoded `open`
- `current_step` — hardcoded `1`
- `is_after_hours` — computed (outside 07:00–17:00 SAST Mon–Fri)
- `nc_number` — generated by Supabase (trigger/sequence)

**Conclusion on clause field:** ISO clause tagging is **optional and recommended**, not blocking. An NC can be fully submitted with zero clauses attached. The field serves as an audit classification aid, not a submission gate.

### Current Clause Suggestion Flow

1. User types description (≥50 chars) → 1.5s debounce fires
2. Calls `suggestIsoClauses()` → invokes `suggest-iso-clauses` Supabase edge function
3. **Edge function is unreachable** — it exists in `supabase/functions/suggest-iso-clauses/` but cannot be deployed via CLI because the project (`bctfghipfzyzzymfnvlv`) is on a separate Supabase account with dashboard-only access
4. On failure/empty result, falls back to `inferClausesFromDescription()` — client-side regex keyword matching across 10 patterns
5. User can also manually type any clause and press Enter, or pick from "Browse all clauses" accordion (all 22 ISO 9001:2015 clauses shown as clickable chips)

**Status:** Client-side fallback is functional and good enough. Edge function would improve accuracy (uses Claude Haiku). Josh needs to deploy `suggest-iso-clauses/index.ts` via the Supabase dashboard for `bctfghipfzyzzymfnvlv` to activate AI suggestions.
